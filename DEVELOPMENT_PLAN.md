# Listen & Pick 开发计划

正式 PRD 见：[docs/PRD.md](docs/PRD.md)

## 产品原则

主张：不背单词，不看字幕，先让孩子听懂英语。

核心体验：孩子在 Pad 横屏上听一句英语短句，从 2 张图中选出匹配图片，答对后自动进入下一题。

## 一期目标

做一个可快速上线、可快速改内容的静态 Web MVP。

- 面向 PC 和 Pad 浏览器访问，优先适配 Pad 横屏。
- 不做登录、不做后台、不做复杂会员。
- 内容先放在本地数据文件，方便快速迭代。
- 单词音频已下载到 `assets/audio/words/`，来源记录在 `data/audio-sources.json`。
- 短句音频当前仍使用浏览器 Web Speech API fallback，后续替换为稳定短句音频文件。
- 每关 15 题，每题 2 张图，答对自动下一题。

## 技术方案

当前版本使用纯前端静态页面：

- `index.html`：页面结构。
- `styles.css`：Pad 优先的响应式样式。
- `app.js`：关卡选择、播放语音、答题、复习逻辑。
- `data/levels.js`：关卡、句子、图片、干扰项、错误标签。

## 内容结构建议

每道题只测试一个语言变量，其余视觉元素尽量保持一致。

示例：

```js
{
  sentence: "The cat is under the chair.",
  skillTag: "position",
  answerId: "cat-under-chair",
  choices: [
    { id: "cat-under-chair", label: "cat under chair", image: "..." },
    { id: "cat-on-chair", label: "cat on chair", image: "..." }
  ]
}
```

常见标签：

- `noun`：名词区分。
- `color`：颜色区分。
- `action`：动作区分。
- `position`：位置区分。
- `number`：数量区分。
- `attribute`：大小、情绪、状态。
