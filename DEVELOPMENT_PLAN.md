# Listen & Pick MVP 开发计划

## 产品原则

主张：不背单词，不看字幕，先让孩子听懂英语。

核心体验：孩子在 Pad 横屏上听一句英语短句，从 2 张图中选出匹配图片，答对后自动进入下一题。儿童学习态尽量不出现额外按钮、文字说明、奖励系统和设置入口。

## 一期目标

做一个可快速上线、可快速改内容的静态 Web MVP。

- 面向 PC 和 Pad 浏览器访问，优先适配 Pad 横屏。
- 不做登录、不做后台、不做复杂会员。
- 内容先放在本地数据文件，方便快速迭代。
- 音频先用浏览器 Web Speech API，后续可替换为音频文件 URL。
- 每关 15 题，每题 2 张图，答对自动下一题。

## 技术方案

当前版本使用纯前端静态页面：

- `index.html`：页面结构。
- `styles.css`：Pad 优先的响应式样式。
- `app.js`：关卡选择、播放语音、答题、复习逻辑。
- `data/levels.js`：关卡、句子、图片、干扰项、错误标签。

部署方式：

- 可直接用任意静态托管：Vercel、Netlify、Cloudflare Pages、飞书妙搭、OSS 静态站。
- 本地预览用 `python3 -m http.server` 即可。

## 版本路线

### V0.1 可体验原型

- 关卡选择页。
- Level 1-3 示例内容。
- 儿童答题页。
- 自动播放英文句子。
- 大号重播按钮。
- 2 选 1 图片答题。
- 答对自动跳下一题。
- 答错自动重播。
- 完成页展示正确率与易错类型。

- 扩充到前 10 关。
- 每题补充 AI 插画提示词。
- 每题补充 `skillTag`，用于分析孩子薄弱点。
- 加入“错题隔 3 题复现”。
- 增加慢速播放逻辑。

### V0.3 上线验证版

- 替换为正式 AI 插画。

- 增加家长入口，查看完成记录和易错标签。

### V0.4 商业化版

- 付费解锁或兑换码。
- 多孩子档案。
- 云端进度同步。
- 家长周报。

## 内容结构建议

每道题只测试一个语言变量，其余视觉元素尽量保持一致。

示例：

```js
{
  sentence: "The cat is under the chair.",
  skillTag: "position",
  answerId: "cat-under-chair",
  choices: [
    { id: "cat-under-chair", label: "cat under chair", image: "..." },
    { id: "cat-on-chair", label: "cat on chair", image: "..." }
  ]
}
```

常见标签：

- `noun`：名词区分。
- `color`：颜色区分。
- `action`：动作区分。
- `position`：位置区分。
- `number`：数量区分。
- `attribute`：大小、情绪、状态。
